Rendering Model
→
The site is fully static (SSG — Static Site Generation). No server-side rendering. All pages are pre-built at deploy time and served from Netlify's CDN. Interactive components (GitHub heatmap, search, skills marquee) hydrate client-side using Astro's client:load directive.

1
Create a new .md file in src/content/posts/
2
Use the frontmatter template from Section 5 (Post Frontmatter Schema)
3
Find an Unsplash image and use the URL format: https://images.unsplash.com/photo-{ID}?w=1400&q=80&auto=format&fit=crop

Create a new directory in src/content/projects/{project-slug}/ with an index.mdx file

Schema Injection Method — Critical Implementation Detail
!
Use set:html — not inline interpolationThe schema must be injected using Astro's set:html directive. Using {JSON.stringify({...})} inside an is:inline script tag renders the literal string as text rather than evaluating it — Google receives unparsable structured data.
<!-- CORRECT — evaluates JSON.stringify() at build time --> <script is:inline type="application/ld+json" set:html={JSON.stringify({...})} /> <!-- INCORRECT — renders literal text "{JSON.stringify({...})}" --> <script is:inline type="application/ld+json">{JSON.stringify({...})}</script>

www.lionelmosley.com — Current Configuration
→
CDN-level redirect via BunnyCDNwww.lionelmosley.com is not a separate deployment. All traffic arriving at this domain is transparently redirected to https://trust-lionel.com. trust-lionel.com remains the canonical URL and primary "url" in the Person schema.
→
sameAs treatmenthttps://www.lionelmosley.com is included in the Person schema sameAs array. This signals to Google that both domains represent the same entity (Lionel Mosley) and consolidates authority under the canonical domain.
